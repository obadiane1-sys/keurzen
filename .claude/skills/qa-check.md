# QA Check — Keurzen

Lis CLAUDE.md et docs/qa/test-checklist.md.

Je veux faire une passe QA sur la feature ou l'écran suivant : $ARGUMENTS

## Ce que j'attends

1. Parcourir le code concerné
2. Vérifier chaque point de la Definition of Done (docs/product/dod.md)
3. Identifier les états manquants (loading / empty / error)
4. Identifier les problèmes de typage ou de logique
5. Identifier les régressions potentielles
6. Donner une liste priorisée des corrections à faire (P0 / P1 / P2)

## Ne pas

- Ne pas refactoriser le code
- Ne pas corriger automatiquement sans lister d'abord les problèmes
- Ne pas toucher aux fichiers hors scope de la feature demandée
